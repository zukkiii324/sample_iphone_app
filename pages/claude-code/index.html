図解：導入〜運用の流れ（5ステップ）

まずは「最小セット」で始めるのが失敗しないコツです。

STEP 1

インストール

公式のQuickstartの手順でCLIを導入します。

npm install -g @anthropic-ai/claude-code
STEP 2

プロジェクト初期化

/init を実行し、プロジェクト用の指示ファイル（CLAUDE.md）を作成します。

STEP 3

依頼を具体化

「目的・制約・完了条件」を明示して依頼すると、手戻りが減ります。

STEP 4

並列実行

複数タスクはGit worktreeを使って分離し、衝突を避けます。

STEP 5

自動化

GitHub Actions連携で、定型作業（要約・レビュー補助など）を自動化します。

図解：依頼文の作り方（テンプレート）

Claude Codeは「良い依頼文」で性能差が出ます。下の順序で書くと安定します。

1. 目的

「何を実現したいか」を1文で宣言する。

2. 制約

使える技術、禁止事項、期限を明示する。

→

3. 完了条件

「どの状態ならOKか」をチェック項目化する。

依頼例 結果の出やすさ 理由
「いい感じに直して」 △ ぶれやすい 目的と完成イメージが曖昧で、修正往復が増えやすい。
「A画面の離脱率を下げる。制約はHTML/CSSのみ。完了条件は3案＋実装1案」 ◎ 安定しやすい 目的・制約・ゴールが明確で、提案の質が上がる。

運用チェックリスト（最初の30日）

  • 初日: Quickstartで導入し、1つの小タスクで試運転する。
  • 1週目: /init とメモリ機能で「プロジェクトの前提」を固定する。
  • 2週目: Git worktreeで並列タスク運用を試し、衝突率を下げる。
  • 3〜4週目: GitHub Actions連携で定型業務を自動化する。

運用のポイント いきなり全業務を任せるより、まずは「議事録要約」「ドキュメント更新」「軽微な修正案作成」など 失敗コストが低い業務から段階導入すると、社内展開がスムーズです。

注目ポイント1

自動判定で拒否された後に再試行を促せる PermissionDenied フックが追加され、運用時の復旧性が上がりました。

注目ポイント2

@ メンション候補に named subagents が表示されるようになり、担当分担がしやすくなりました。

注目ポイント3

長時間セッションでのキャッシュ・メモリ関連の修正が複数入り、安定性が改善されています。
